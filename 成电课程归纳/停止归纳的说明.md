# 采用归纳的方法

1. 摘抄定义概念
2. 给出自己对内容的概括或者理解/记忆模型
3. 给出自己的一些推论等
4. 忽略无关紧要的内容

# 停止归纳的原因

1. 太耗时，数学公式太多，完整的MD语法描述太复杂
2. PPT足够精简，摘抄内容太多且太冗余
3. 如果不摘抄，只记录原文位置，将来查看翻找太麻烦
4. 摘抄和公式描述的复杂性打断思路

60%~80%以上的内容，为PPT原文，也就是摘抄，远不如使用MarginNote方便。
> 之所以不使用MarginNote，是因为其为第三方软件，鬼知道什么时候就不更新了
> 而且我记录在上的内容也够多了，软件都变卡了，且内容导出不便
> 最重要的是，软件不咋好用了，对于删除、回收站、备份、恢复竟然有时会出现混乱
> 所以我会使用MD来记录，也不足为奇，完全自主可控，随意备份，样式精美

现在使用MD做读书笔记的最大问题就是摘录不便，做完全的技术博客还好，从无到有不涉及到摘录。

所以停止使用MD作为读书笔记，新方法暂定为直接添加PPT/PDF文件批注
